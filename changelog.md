# 修改記錄 (Changelog)

## 初始開發 

### 1. 建立前端架構與介面 (index.html, style.css)
- 完成基本的手機優先 (Mobile-First) 的響應式設計。
- 採用簡單且具現代感的 UI 避免過度複雜的樣式，以符合「保持簡單，不要過度設計」的要求。
- 新增：音檔載入區、播放控制區 (時間軸、播放速度、跳轉等)、標記與清單區塊。

### 2. 實作音效功能 (script.js)
- **播放控制**：包含時間軸 (可拖拉跳轉進度)、音量設定 (預設 0.5)、播放速度調整 (0.75, 1, 1.25, 1.5)、快轉倒轉 (3, 5, 10, 30 秒功能)，以及可針對 (時:分:秒) 做特定跳轉的功能。
- **標記區間**：在按下「設定開始標記」及「設定結束標記」時，會自動暫停音軌，並且可供輸入區間名稱後存入內部的資料陣列。

### 3. 實作資料維護功能
- **操作方式**：可匯出目前的區間清單為 JSON 格式 (標籤保留為：名稱、開始、結束)，以音檔名作為檔名前綴儲存。
- **開啟區間資料**：可讀取預存的 JSON 資料檔直接還原進清單內。
- **開啟空白表單**：實作重新清除與初始化的按鈕。

### 4. 實作實體剪輯與匯出功能 (JSZip + Web Audio API)
- 透過 JSZip 與 WebAudio API 把當下已選定的片段切割成獨立的 Wav 音檔。
- 打包成一個被統一名稱為「原音檔名稱.zip」的壓縮檔，壓縮檔內部包含了「原音檔名稱/區間名稱.wav」的所有檔案，確實達到「以當前音檔名生成資料匣並存於其中」的限制要求，這讓系統不依賴後端即可有很強大的跨平台行動裝置可用性。

### 5. 新增播放進度條 (時間軸)
- 在播放控制區新增一個可以拉動的狀態條 (Progress Bar)。
- 支援透過拉動、點擊進度條來即時改變播放時間的跳轉。

語言與風格已全數修正為繁體中文及台灣常用語氣與模式。

### 6. 優化與功能調整 (最新更新)
- **移除剪輯功能**：拔除原本的 JSZip 相關獨立音檔剪接下載邏輯，讓程式碼與介面更輕量專注於標記。
- **改進跳轉播放**：在「跳轉至特定時間」點擊跳轉後，將立刻依據指定的時間自動恢復播放音檔。
- **介面優化**：「跳轉至特定時間」中的所有數字輸入框，以及時間輸入框的數值，現在皆改為「置中對齊」。
- **手動標記設定**：標記區間的「開始時間」和「結束時間」已顯示為輸入框型態，支援按鈕帶入時間，也能直接手動輸入數字與時間字串 (如 00:01:05 或 65)。
- **支援儲存 MD 檔**：存入第一筆區間資料後，會自動以目前的「音檔名稱」產生 Markdown (`.md`) 檔並觸發下載/儲存事件。(注意：由於為全前端靜態網站，無法直接經伺服器存入 Google 雲端硬碟，建議使用者於彈出存檔視窗時，手動選擇並存於本機端的 Google 雲端硬碟自動同步資料夾以達成雲端備份效果)。
- **資料表編輯功能**：已設定的區間資料表新增「編輯」按鈕，點擊後會彈出視窗讓使用者可直接手動更改該區段的名稱及起訖時間。
- **批次與空白表單**：點擊「開啟空白資料表」時，現在會跳出專屬的彈出式視窗，包含一般的區間資料欄位可供直接填寫外，也頂部提供「文字貼上區」，支援透過直接從 Excel / 純文字複製並以 Tab 等符號分隔的多筆資料快速「解析」並「套用」，提升大量資料的操作體驗。
